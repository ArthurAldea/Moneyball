# Phase 06.1 Plan 01: Understat Scraper Foundation Summary

**One-liner:** Wave 0 TDD stubs + UNDERSTAT_LEAGUE_NAMES config + generalized per-league `scrape_understat_league()` with 7-day cache, covering 5 leagues x 2 seasons.

## What Was Built

### Task 1: Wave 0 test stubs (commit 3541001)
Added 8 Wave 0 stubs across test_scraper.py and test_merger.py defining contracts before implementation:

- **test_scraper.py** (4 stubs): `test_understat_league_columns`, `test_understat_cache_naming`, `test_understat_cache_hit`, `test_run_understat_all_leagues`
- **test_merger.py** (4 stubs): `test_attach_understat_exact_match`, `test_attach_understat_fuzzy_match`, `test_attach_understat_unmatched`, `test_per90s_xg_xa`

Stubs fail for correct reasons: scraper stubs at `ImportError` (function absent), merger stubs at `ImportError` for `attach_understat_xg` (Plan 02 implements), `test_per90s_xg_xa` at assertion (config not yet updated).

### Task 2: config.py additions (commit f466671)
Three additions to config.py:
- `UNDERSTAT_LEAGUE_NAMES`: Maps 5 league keys → Understat slugs (`EPL`, `La_liga`, `Bundesliga`, `Serie_A`, `Ligue_1`)
- `"xG"` and `"xA"` appended to `SUM_STATS`: cross-season summation in `_aggregate_fbref_seasons` (NaN until Plan 02 calls `attach_understat_xg`)
- `"xG"` and `"xA"` appended to `PER90_STATS`: `compute_per90s()` auto-produces `xG_p90`/`xA_p90` without merger.py changes

`test_per90s_xg_xa` transitions from AssertionError to PASSED after this task.

### Task 3: scraper.py implementation (commit 3b040b1)
Four changes to scraper.py:
- Added `UNDERSTAT_LEAGUE_NAMES` to the config import block
- Added `_fetch_understat_league(league, season_year, season_label)` — async, generalized from EPL-only `_fetch_understat_season`, uses `UNDERSTAT_LEAGUE_NAMES[league]` slug lookup
- Added `scrape_understat_league(league, season_year, season_label)` — cache-first with `cache/understat_{league}_{season_label}.csv` naming, 7-day TTL, returns empty DataFrame on failure
- Replaced deprecated `run_understat_scrapers() -> {}` stub with real implementation covering all 5 leagues x 2 seasons, returns `{league: {season_label: DataFrame}}`

All 4 Wave 0 scraper tests now PASS. `scrape_understat_season()` preserved for backward compat.

## Decisions Made

1. **Cache key convention for new scraper**: `understat_{league}_{season_label}` (e.g. `understat_EPL_2024-25.csv`) — differs from legacy EPL-only `understat_{season_label.replace('-','')}` format. Both coexist without conflict.
2. **`_fetch_understat_league` is additive**: `_fetch_understat_season` kept intact for backward compat; both functions exist independently.
3. **`run_understat_scrapers` return structure**: `{league: {season_label: DataFrame}}` — matches what `attach_understat_xg` in Plan 02 will consume as its `understat_data` argument.
