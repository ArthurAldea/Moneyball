## Summary

The `understat` Python library is already installed (`requirements.txt`) and was previously used to scrape EPL-only xG/xA data. Existing cache files (`cache/understat_202324.csv`, `cache/understat_202425.csv`) cover EPL only — they must be replaced with per-league files covering all 5 leagues. The old scraper used `get_league_players("EPL", season_year)` with a single cache file per season; the new scraper must call `get_league_players` for each of the 5 leagues and write separate cache files using the convention `cache/understat_{league}_{season}.csv`.

Understat covers all 5 target leagues (EPL, La Liga, Bundesliga, Serie A, Ligue 1) and has data since 2014-15. The API endpoint uses internal league-slug names (`EPL`, `La_liga`, `Bundesliga`, `Serie_A`, `Ligue_1`). The Python library's `to_league_name()` function maps common variants, but case-sensitive exact keys work directly. Rate limiting is not a known issue with Understat (it is a small async library making single-endpoint JSON requests), but polite delays are still appropriate.

The merge strategy is straightforward: join Understat xG/xA onto the FBref DataFrame by `Player` name using the same normalized-name + rapidfuzz pipeline already used for Transfermarkt matching. Club cross-check is advisable for disambiguation. After joining, derive `xG_p90` and `xA_p90` from `xG / (Min/90)` using Understat's raw `Min` field to validate denominator, but use the FBref `Min` column (already summed across seasons in the merged DataFrame) as the canonical denominator when computing per-90 rates to keep denominators consistent.

**Primary recommendation:** Add `scrape_understat_league()` function to `scraper.py` with per-league cache naming; add `attach_understat_xg()` to `merger.py` for the join; update `config.py` pillar weights to blend `xG_p90` and `xA_p90` into Attacking and Creation pillars.

### Understat League Name Mapping

The `understat` library's `to_league_name()` has a lookup table using lowercase snake-case keys (`epl`, `la_liga`, etc.). For clarity and safety, the new scraper should use the internal slug names directly:

| Project League Key | Understat slug | URL produced |
|--------------------|---------------|--------------|
| `EPL` | `EPL` | `understat.com/getLeagueData/EPL/{year}` |
| `LaLiga` | `La_liga` | `understat.com/getLeagueData/La_liga/{year}` |
| `Bundesliga` | `Bundesliga` | `understat.com/getLeagueData/Bundesliga/{year}` |
| `SerieA` | `Serie_A` | `understat.com/getLeagueData/Serie_A/{year}` |
| `Ligue1` | `Ligue_1` | `understat.com/getLeagueData/Ligue_1/{year}` |

This mapping must live in `config.py` as `UNDERSTAT_LEAGUE_NAMES`.

### Pattern: Per-League Understat Scraper

```python
# Source: existing _fetch_understat_season() in scraper.py — generalised

async def _fetch_understat_league(league: str, season_year: int, season_label: str) -> pd.DataFrame:
    """Fetch Understat player data for one league+season."""
    import aiohttp
    from understat import Understat
    from config import UNDERSTAT_LEAGUE_NAMES

    understat_slug = UNDERSTAT_LEAGUE_NAMES[league]

    async with aiohttp.ClientSession() as session:
        understat = Understat(session)
        players = await understat.get_league_players(understat_slug, season_year)

    rows = []
    for p in players:
        rows.append({
            "Player":  p.get("player_name", ""),
            "Squad":   p.get("team_title", ""),
            "Pos":     _map_understat_pos(p.get("position", "")),
            "Min":     float(p.get("time", 0) or 0),
            "xG":      float(p.get("xG", 0) or 0),
            "xA":      float(p.get("xA", 0) or 0),
            "season":  season_label,
        })
    return pd.DataFrame(rows)


def scrape_understat_league(league: str, season_year: int, season_label: str) -> pd.DataFrame:
    """Scrape Understat xG/xA for one league+season with 7-day cache."""
    cache_key = f"understat_{league}_{season_label}"
    path = _cache_path(cache_key)
    if _is_fresh(path):
        return pd.read_csv(path)
    df = asyncio.run(_fetch_understat_league(league, season_year, season_label))
    if not df.empty:
        df.to_csv(path, index=False)
    return df


def run_understat_scrapers(leagues=None, seasons=None) -> dict:
    """
    Scrape Understat for all leagues × seasons.
    Returns {league: {season: DataFrame}}.
    """
    from config import FBREF_LEAGUES, FBREF_SEASONS, SEASONS, UNDERSTAT_LEAGUE_NAMES
    if leagues is None:
        leagues = list(FBREF_LEAGUES.keys())
    if seasons is None:
        seasons = FBREF_SEASONS  # ["2023-24", "2024-25"]

    results = {}
    for league in leagues:
        results[league] = {}
        for season_label in seasons:
            season_year = SEASONS[season_label]
            df = scrape_understat_league(league, season_year, season_label)
            results[league][season_label] = df
    return results
```

### Pattern: attach_understat_xg() in merger.py

The join strategy reuses the existing `normalize_name` + rapidfuzz pattern. Key decisions:

1. **Join on normalized player name only (Pass 1 + Pass 2)** — skip Pass 3 (club cross-check) for the primary join because club names differ significantly between Understat and FBref (e.g. Understat uses "Wolverhampton Wanderers", FBref uses "Wolves"). Only add club cross-check as Pass 3 if multiple players have identical names.

2. **Aggregate Understat across seasons before joining** — sum `xG` and `xA` across seasons using the same season scope as FBref data, then join onto the already-aggregated FBref DataFrame.

3. **Use FBref `Min` as denominator** — the merged FBref DataFrame already has `Min` (summed across seasons). Use this to derive `xG_p90 = xG / (Min/90)` for consistency with all other `_p90` derivations in `compute_per90s()`.

### Pattern: Pillar Weight Updates in config.py

## Common Pitfalls

### Pitfall 1: Understat Multi-Club Player Squad Names
**What goes wrong:** Understat records a player's last club only (e.g. "Liverpool" after a mid-season transfer), not the combined "Club1,Club2" string FBref uses. FBref has a "2 Clubs" summary row; Understat has a single row with the final club.
**Why it happens:** Different data model — Understat shows the player's current club at time of data fetch.
**How to avoid:** Join on player name only, not player+club. For the rare name collision, use club cross-check as Pass 3 only (lower threshold, not primary filter).
**Warning signs:** Players who transferred mid-season get NaN xG/xA after join — inspect unmatched count post-join.

### Pitfall 2: Understat Club Names vs FBref Squad Names
**What goes wrong:** Understat uses full English club names; FBref uses abbreviated or accented names.

| Understat | FBref |
|-----------|-------|
| Wolverhampton Wanderers | Wolves |
| Manchester United | Manchester Utd |
| Newcastle United | Newcastle Utd |
| Tottenham | Tottenham Hotspur |
| Nottingham Forest | Nottingham Forest (same) |
| Ipswich | Ipswich Town |
| Leicester | Leicester City |
| West Ham | West Ham United |

**How to avoid:** Do NOT use exact club name matching. Use rapidfuzz on club names only as a soft tie-breaker (Pass 3), not as a blocker.

### Pitfall 3: Season Year vs Season Label
**What goes wrong:** Understat uses the start year as an integer (`get_league_players("EPL", 2024)` = 2024-25 season). The project uses `"2024-25"` labels; `SEASONS["2024-25"] = 2024` maps them.
**How to avoid:** Always pass `SEASONS[season_label]` as the `season_year` argument to the Understat library.

### Pitfall 4: aiohttp Event Loop Conflict
**What goes wrong:** If called from within an already-running async context (e.g. Streamlit), `asyncio.run()` raises `RuntimeError: This event loop is already running`.
**How to avoid:** Understat scraping must happen in `scraper.py` entry point (offline, before app.py runs). Never call `run_understat_scrapers()` from inside Streamlit. Same pattern as existing FBref scraper.

### Pitfall 5: Unmatched Players Silently Scoring 0
**What goes wrong:** If `xG_p90` is `NaN` for unmatched players, `_score_group` fills NaN with 0 before MinMaxScaling — meaning unmatched players score minimum on xG pillar component, which artificially deflates their score.
**How to avoid:** After joining, log the match rate. If below ~80%, investigate. FBref Min-filter (900 min) means many fringe players who exist in FBref won't exist in Understat (fewer appearances). NaN is acceptable for such players.

### Pitfall 6: asyncio.run() vs loop().run_until_complete()
**What goes wrong:** The existing `_fetch_understat_season` uses `asyncio.run()` — this is correct for scraper.py. The old code in the file docstring references `nodriver`'s event loop (`uc.loop().run_until_complete()`). Don't mix these.
**How to avoid:** Use `asyncio.run()` for all Understat async calls, consistent with the existing `scrape_understat_season()` pattern.

## Pipeline Integration Point

The Understat scrape must slot into the existing pipeline in `app.py` / `run_scoring_pipeline()`:

```
Current pipeline:
  run_fbref_scrapers() → run_tm_scrapers() → build_dataset() → compute_scout_scores() → ...

New pipeline:
  run_fbref_scrapers() → run_understat_scrapers() → run_tm_scrapers()
                       ↓
                build_dataset(fbref_data, tm_data, understat_data)
                  └── attach_understat_xg() called inside build_dataset
                       ↓
                compute_per90s()  [automatically derives xG_p90, xA_p90]
                       ↓
                compute_scout_scores()  [now sees xG_p90, xA_p90 in pillar dicts]
```

`build_dataset()` in merger.py needs a third parameter `understat_data` (or it's a separate step), and `attach_understat_xg()` is called after `compute_per90s()`. The `run_scoring_pipeline()` entry point in scorer.py must also be updated to call `run_understat_scrapers()` and pass the result through.

**Alternative (simpler):** `attach_understat_xg()` is called from within `build_dataset()` before `compute_per90s()`, injecting raw `xG` and `xA` columns onto the merged DataFrame. Then `compute_per90s()` naturally produces `xG_p90` and `xA_p90` since those are listed in `PER90_STATS`. This keeps the pipeline linear and `build_dataset()`'s signature change minimal.

## Open Questions

1. **Should `attach_understat_xg()` be called before or after `compute_per90s()`?**
   - What we know: Adding `xG`/`xA` to `PER90_STATS` means `compute_per90s()` handles derivation automatically — cleaner.
   - What's unclear: If `attach_understat_xg()` runs before `compute_per90s()`, the raw `xG`/`xA` columns must exist on the DataFrame at that point.
   - Recommendation: Call `attach_understat_xg()` inside `build_dataset()` just before `compute_per90s()`. This is cleaner and keeps all per-90 derivation in one place.

2. **Do we sum Understat xG/xA across seasons before joining?**
   - What we know: FBref data is aggregated across 2 seasons. Understat data is per-season. A player who appeared in both seasons should have their xG/xA summed across both seasons.
   - Recommendation: Yes — `attach_understat_xg()` should aggregate `understat_data` across seasons per player per league before joining, using `groupby("Player").agg({"xG": "sum", "xA": "sum"})`.

3. **What match rate should trigger a warning?**
   - Recommendation: Log a warning if fewer than 70% of FBref players receive an Understat xG match for a given league. This accounts for the 900-minute filter (some FBref players won't appear in Understat).
